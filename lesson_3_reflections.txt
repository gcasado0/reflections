When would you want to use a remote repository rather than keeping all your work
local?

    Cuando se quiere compartir del desarrollo con otra persona y tambien para tener una copia de los fuentes.

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

   Porque si estoy trabajando un algun cambio no quiero que me modifiquen los archivos todo el tiempo, con cambios de otras personas. 

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

   El branch solo existe dentro de un repositorio, y sirve para tener una version distinta el master. Hacer un prueba, solucionar un problema o agregar una nueva funcion.
   El clonar sirve para bajar a la pc un repositorio de un sitio como github o bitbucket. Tambien se puede clonar un repositorio local, para empezar a lo mejor un nuevo sistema con la base de otro.
   El fork es para clonar un proyecto en github y empezar a modificarlo uno mismo sin afectar el original.


What is the benefit of having a copy of the last known state of the remote
stored locally?

    Te permite hacer cambios y trabajar sin conexion, y luego sincronizar tus cambios con el resto del equipo.

When would you want to make changes in a separate branch rather than directly in
master?  What benefits does each approach have?
